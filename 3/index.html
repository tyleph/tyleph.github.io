Part A.2: Recovering Homographies

To align the images, I needed to compute the homography matrix $H$, a 3x3 matrix with 8 degrees of freedom that transforms points from one image plane to another ($p' = Hp$). I manually selected at least 4 correspondence points between each pair of images. These points create an overdetermined system of linear equations of the form $Ah=b$, which I solved using least squares to find the 8 unknown values of $H$.

Part A.3: Image Rectification

Rectification of a Painting

The results from bilinear interpolation are noticeably smoother and have fewer artifacts compared to nearest-neighbor interpolation, which produces a blockier result. This is because bilinear interpolation considers a weighted average of the four nearest pixel values, creating a more gradual transition of colors. The trade-off is that bilinear interpolation is slightly slower due to the additional calculations, while nearest neighbor is faster but lower quality.

Part A.4: Mosaicing

With all the components working, I combined everything to create panoramic mosaics. The process involves:

  1. Manually selecting correspondence points between two images.
  2. Computing the homography matrix $H$ to transform one image into the coordinate system of the other.
  3. Warping the image using the computed homography and bilinear interpolation.
  4. Blending the warped image and the base image. I used a simple weighted average in the overlapping region to create a smooth transition and avoid harsh seams.
